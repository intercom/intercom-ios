IntercomSessionListener Protocol Reference

Conforms to NSObject
Declared in Intercom.h

Overview

Notifications thrown by the SDK when the SDK window is displayed and hidden. These notifications are fired only when there is a change in the state of the SDK’s UI: when a user receives a message for instance, willShow and didShow notifications will be fired accordingly when the Intercom Notification (chat head) is presented.

Once the user taps on the chat head, the message is presented in your app. It will be presented covering the entire screen, but no notifications will be thrown here as the SDK has already been visible.
